# 奇石鉴赏智能体

> 让一块石头变成短片的视觉主角，让观众看完啧啧称奇。

## 我能为你做什么

上传一张奇石图（雨花石、灵璧石、太湖石、戈壁石、寿山石、化石、玉璞等任意品类），生成一支约 1 分钟的鉴赏短片素材包：

- **石头是绝对主角**：超过一半镜头聚焦石头本身——纹理、光泽、形态、肌理的特写与微距。让石的"奇"被一寸寸放大出来
- **人物是凝视者**：一位现代中国知性雅人（30-45 岁，男女皆可），充当"鉴赏者视角"——把玩、端详、对光举起、轻轻摩挲，让观众通过 ta 的眼睛重新发现这块石
- **观众体感**：第一眼看完想说"这石头真好"，第二眼想再放大看看纹理
- **风格可选**：启动时选择写实纪录风 / 梦幻意境流 / 东方诗意流；默认写实纪录风。任何风格下人物都保持现代服饰，石头始终是视觉主角

## 你会拿到什么

- `final_report.md` —— 完整故事板（含每镜的可灵/Seedance 视频生成 Prompt）
- `character_reference_sheet.png` —— 鉴赏者定档立绘（保证人物镜头一致性）
- `shot_01.png` ~ `shot_N.png` —— 每镜静帧（默认 8 张，~1 分钟成片）
- `character_bible.json` / `storyboard.json` —— 结构化数据，便于二次开发

## 使用方法

1. 在对话中上传一张奇石图片
2. 在弹出的表单里选择短片风格；也可输入标题、想要镜数（默认 8 镜；写"做 16 镜头版本"切到 16）、特定氛围倾向
3. 等待 ~6 分钟（默认 8 镜）/ ~12 分钟（16 镜）
4. 打开生成的 `final_report.md`，按里面的 Prompt 把图变成视频

## 产品边界

- **每镜必须出现石头或暗示石头**——石头是叙事和视觉的中心，人物镜头不能多于石头镜头
- 镜头组成大致比例：**石头特写/微距 ~50%** + 人物与石互动（鉴赏、把玩、对光）~35% + 环境留白 ~15%
- 不做古装/武侠/玄幻/奇幻战斗。写实纪录风禁超现实光晕、魔法粒子、梦境氛围；梦幻意境流可保留适度光晕、雾气、漫光；东方诗意流强调宋画留白、水墨灰青与文人器物。任何风格下人物都穿现代棉麻/真丝/针织等当代服饰
- 不模仿王家卫式克制、黑泽明式凝重——基调**明快通透、轻盈愉悦**
- 仅适配 Web 客户端

## 部署须知（运维 / 自部署 ArkLoop 时）

本智能体调用 4 个外部模型（OpenRouter 凭据为必需，其他为可选）：

| 阶段 | 模型 selector | 凭据要求 |
|---|---|---|
| 图像理解 | `qwen3.6-plus` | 阿里通义凭据 + 一条 `model=qwen3.6-plus` 的 route |
| 故事 / Bible / 分镜 | `deepseek-v4-flash`（生产可升档到 `deepseek-v4-pro`） | DeepSeek 凭据 + 对应 model 的 route |
| 图像生成 | `openai/gpt-5-image-mini` | **必需**：OpenRouter 凭据 + `model=openai/gpt-5-image-mini` 的 image route |

若 OpenRouter 凭据未配置，本智能体启动后会在第一次图像生成时硬失败并提示。建议在启用本 persona 前先在 ArkLoop 中配好该路由。

默认 8 镜头（成本 ~$0.4 / run，~6 分钟）。需要 16 镜头版本在用户文本里写「做 16 镜头版本」即可。调试时可设环境变量 `ARKLOOP_YUHUA_SHOT_COUNT_DEBUG=4`（worker 容器）跑 4 镜头快速验证。
