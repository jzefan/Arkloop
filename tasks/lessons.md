# Lessons

## 交互 / 自主性

### 2026-05-31 — 不要过度向用户确认
- **纠正**："不要总让我确定，除非需求上需要确定的。"
- **背景**：在已获明确指令（"先做2"）后，仍多次停下来等确认、反复汇报中间状态。
- **规则**：
  - 需求已明确时，**直接做到底再汇报结果**，不要逐步征求许可。
  - 只有当存在"会改变产出方向、且无法从代码/上下文/合理默认推断"的决策点时，才用 AskUserQuestion。
  - 后台任务（下载/编译/测试）跑着时不要每隔一步就汇报；等结果出来一次性给结论。
  - 提交（commit/push）仍遵循 git 规则：仅在用户要求时执行——但这不等于"每件事都要问"。
